# SewageMachine

基于 STM32F1 的污水清洁设备控制固件项目，集成涂鸦蓝牙通信、行走电机控制、抽水电机控制、电池监测、姿态检测和定时清扫功能。

本仓库保存的是设备主控 MCU 固件源码，适合通过 Github 进行版本管理、协作开发和问题追踪。

## 项目简介

该项目采用典型的 STM32 裸机 `super loop` 架构：

- 中断负责基础事件采集，如串口收字节、ADC DMA 完成、1ms 定时节拍
- 主循环负责按键状态机、涂鸦协议解析、电机状态机、ADC 平均、定时到期回调等业务逻辑
- 电机动作由全局状态变量和非阻塞状态机驱动，不依赖 RTOS

当前主要功能包括：

- 本地按键开关机
- 涂鸦 App 开关机与模式控制
- 手动控制前进、后退、左转、右转
- 自动清扫模式
- 爬壁模式
- 定时清扫
- 电池电压与电机电流检测
- 蓝牙 DP 点上报与下发控制

## 硬件与软件栈

- MCU：`STM32F103xE`
- 外设驱动框架：`STM32 HAL`
- 工程方式：
  - `EIDE`
  - `Keil MDK-ARM`
- 配置来源：`STM32CubeMX`
- 蓝牙协议：`Tuya MCU SDK`
- 姿态传感器：`QMI8658`

## 工作模式说明

### 1. 逻辑关机等待态

设备上电后，MCU 会先完成基础初始化，但不会立即进入正式运行状态，而是处于“逻辑关机等待态”。

进入工作状态的方式有两种：

- 本地按键短按开机
- 涂鸦 App 下发开机命令

该阶段下：

- 所有电机保持关闭
- 电量灯关闭
- 蓝牙协议仍持续工作

### 2. 手动控制模式

涂鸦 App 可直接下发以下动作：

- 前进
- 后退
- 左转
- 右转

这些动作通过 `MotorCtrl_SetAction(...)` 直接驱动电机输出。

### 3. 自动清扫模式

自动清扫采用内部非阻塞状态机实现，典型流程包括：

- 直行
- 转向
- 短距离前进
- 再次转向
- 循环执行

### 4. 爬壁模式

爬壁模式结合以下输入判断：

- 碰撞输入
- IMU 姿态角

典型流程包括：

- 向墙面行进
- 检测到爬壁姿态
- 持续爬壁
- 后退下墙
- 转向后继续下一轮

### 5. 定时清扫模式

定时清扫通过软件定时器实现。

当前代码逻辑为：

- 定时功能会跟随当前工作模式
- 如果当前处于爬壁模式，则启动定时后继续保持爬壁模式
- 其他情况下，定时默认按自动清扫模式运行

定时到期后：

- 停止全部电机
- 工作状态回到待机
- 向 App 上报状态更新

## 主要控制流程

### 开机流程

1. HAL 初始化
2. 时钟配置
3. 初始化 GPIO / DMA / UART / I2C / TIM / ADC
4. 启动 PWM 与 1ms 定时中断
5. 初始化软件定时器、电机控制、蓝牙协议、按键模块
6. 进入逻辑关机等待态
7. 收到开机动作后，初始化 IMU、ADC DMA 等运行资源
8. 进入主循环

### 主循环内容

主循环中主要执行：

- 按键任务
- UART1 文本命令处理
- 涂鸦蓝牙协议服务
- 蓝牙状态轮询
- 电机任务状态机
- 软件定时轮询
- 碰撞输入扫描
- ADC 平均与换算
- 电量上报

## 开发注意事项

- 本项目为裸机工程，不使用 RTOS
- 业务逻辑中大量使用全局状态变量，修改功能时需要同时关注：
  - 涂鸦协议层状态变量
  - 电机业务层实际执行逻辑
- `CubeMX`、`EIDE`、`Keil` 会生成大量本地构建和缓存文件，不建议提交到仓库
- 仓库中包含 STM32 HAL、CMSIS、Tuya SDK 等第三方代码，使用时请注意其许可证要求
